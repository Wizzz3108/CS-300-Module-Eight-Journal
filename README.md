# CS-300-Module-Eight-Journal

# For this assignment, I was solving the problem of helping academic advisors efficiently manage and access course information without having to manually sort or search through it. The goal was to take a raw data file and organize it in a way that allows for fast searching and clean output. I approached this by using a binary search tree since it naturally keeps data sorted and allows for efficient lookups when implemented correctly. This project really showed me why understanding data structures matters, because the way data is stored directly impacts how efficient and scalable the program is.

# While working through the project, I ran into a few roadblocks, mainly with the binary search tree logic and making sure the search function worked correctly. Small mistakes in traversal direction caused bigger issues, but debugging and slowing down to follow the logic step by step helped me fix them. This project expanded how I approach designing software by making me focus more on structure and planning before coding. It also improved how I write programs by pushing me to keep my code readable, organized, and easier to maintain, instead of overcomplicating things or writing messy logic.
